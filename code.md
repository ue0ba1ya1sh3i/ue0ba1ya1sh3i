## Basics
- Develop as SaaS
- Insert a blank line between logical sections
- Always end with a newline
- For unclear parts, write comments in Japanese explaining the "why"
- Abbreviate official names if they are too long

## Code
- Keep `export default` as-is
- Do not add a trailing comma to JSON objects
- Do not use colons
- Place static files in `public/files/`
- Use aliases if path definitions become long

## Others
- Constants such as `.env` should be written in `UPPER_UNDERSCORES`
- Project names should be in `kebab-case`
- File and folder names should be in `snake_case`
